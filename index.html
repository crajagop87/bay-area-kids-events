Built by a Palo Alto parent who wanted a faster way to find things to do with the kids — no newsletter subscriptions, no app downloads, just a clean feed of what's actually happening nearby.

A volunteer project. Questions or ideas? Hit Feedback.

Coverage

Events within ~20 miles of Palo Alto, pulled from public library systems (BiblioCommons + LibCal), Eventbrite, local aggregators (Funcheap, Bay Area Kid Fun, 510 Families), and specific venues (Hidden Villa, Hiller Aviation, Cantor Arts, Exploratorium, Palo Alto Junior Museum, CuriOdyssey, Midpeninsula Open Space). New: Sunnyvale Public Library and San Mateo Public Library via LibCal widget.

Scrape schedule

Events are pulled every day at 6 am PT. The timestamp in the header shows when the last pull completed.

        When

        • All — no date filter; shows everything in the scraped window (45 days).
        • Today — events happening on today's date only.
        • Next week — rolling 7-day window from today.
        • Custom — pick a start and end date. End date can't be set before start date.

        Where

        • Palo Alto — haversine distance filter centered on Palo Alto. Default radius 5 mi.
        • SF — centered on downtown San Francisco.
        • 📍 Custom — opens the map so you can drag the pin anywhere in the Bay Area.
        • All areas — no location filter; every scraped event is shown.
        • Width slider — sets the diameter of the coverage circle; drag freely or snap to tick marks. The 📍 Custom button opens a map showing the circle; width set in the map transfers back when you tap "Use this width."

        Age

        • Dual-handle slider for ages 0–18. Events are shown if their inferred age range overlaps your selection at all — not just exact matches.
        • Age groups are inferred from event titles and descriptions using keyword patterns (e.g. "toddler", "ages 3–5", "elementary").

        Search

        • The search bar filters by title, venue, city, and tags simultaneously — tags are indexed but not shown on cards.
        • Try: "storytime", "science", "free", "farm", "ice cream", "music", "drop-in".

        Versioning strategy

        • Format: vYYYY.MMDD.N — date is the push day; N is a manual counter that resets to 1 on each new date and increments with every FE push.
        • FE version — increment N in FE_VERSION before each push.
        • BE version — scraper run timestamp in footer; increments automatically each daily 6 am scrape.
        • Git log is authoritative ordering within a day if N goes out of sync.

        Architecture

        • No login, no tracking, no ads. One URL, shareable with filter state in query params.
        • Static frontend on GitHub Pages — zero server cost, zero maintenance surface.
        • Python scraper runs daily on GitHub Actions, writes events.json + health.json.
        • Filter state persists in localStorage; URL params allow sharing a filtered view.

        UX decisions

        • Defaults to Palo Alto · 5 mi · ages 0–5 · next 7 days — common case, not a blank slate.
        • Radius is continuous (0.1 mi steps) — families think "how far am I willing to drive."
        • Cards fade on filter change — avoids jarring pop of immediate replacement.
        • Map modal shows live coverage circle while dragging + dynamic suburb label.
        • Sticky filter summary + floating date label when scrolled.

        Visual design

        • Bricolage Grotesque for headers; Plus Jakarta Sans for body.
        • Warm off-white (#fdf8f3) — easier on the eyes than pure white for list browsing.
        • Orange accent. Free events: green left-border. Tags: neutral grey (color by category added noise).
        • Skeleton shimmer on load communicates structure before content arrives.

        Data & scraping

        • Adult-noise filter strips wine tastings, networking, comedy, financial seminars pre-frontend.
        • Age groups inferred by regex chain: babies → toddlers → preschool → kids → teens → family.
        • Eventbrite scraped across 30+ Peninsula/Bay Area cities; deduped by title+date+city hash.
        • BiblioCommons API: Palo Alto, SMCL, SCCL, SJPL, East Bay. LibCal ICS: Mountain View. LibCal widget: Sunnyvale, San Mateo City.
        • All datetimes stored as true UTC; displayed in America/Los_Angeles via Intl.DateTimeFormat. Post-scrape sanity check flags PT start times 1–5 AM.
        • Tags: closed canonical taxonomy (35 tags in tagger.py) + SYNONYM_MAP. Hidden from cards, indexed for search.
        • UX revert: git checkout <hash> -- index.html. Named snapshots: git tag -l 'ux/*'.

        Every push, newest first. Each date is a daily release; individual builds listed within.
